## üéØ Implementation Guidelines

### DO ‚úÖ

- ‚úÖ Link to blog posts only when they **directly support** the surrounding content
- ‚úÖ Use **descriptive anchor text** that hints at the link destination
- ‚úÖ Spread links across **different sections and pages** (don't cluster)
- ‚úÖ Link once per concept (don't repeat the same post multiple times on one page)
- ‚úÖ Choose blog posts that **demonstrate specific TCNA knowledge**
- ‚úÖ Test links before publishing to ensure they work

### DON'T ‚ùå

- ‚ùå Force links into sentences where they feel awkward
- ‚ùå Use generic anchor text like "click here" or "learn more"
- ‚ùå Link multiple times to the same blog post on the same page
- ‚ùå Add links to pages that don't naturally support them (about, contact, pricing)
- ‚ùå Use links as a replacement for clear, complete content on the page
- ‚ùå Over-link (max 3-4 per page, 1-2 per section)

---

## üîç Link Quality Checklist

Before adding each link:

- [ ] Does this link **directly answer** a question in the surrounding text?
- [ ] Would a reader **benefit** from this context, or does it feel promotional?
- [ ] Is the anchor text **descriptive** (not "click here" or "learn more")?
- [ ] Have I already **linked to this post** on this page?
- [ ] Does the blog post **add technical depth**, not just repeat the page?
- [ ] Is the blog post **relevant to the page topic** (not off-topic)?
- [ ] Are there **fewer than 4 links** on this page total?
